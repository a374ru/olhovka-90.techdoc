# Проект технической документации хозяйства Ольховка 90

**Проект монтируется на платформе HUGO языка GO.**

 Генерируется статический сайт с инструкциями о пользовании приборами хозяйством. Данный сайт располагается на двух хостинга `GitHub` и `Firebase`. Адреса данных проектов представлены ниже:

1. https://home.a374.ru
2. https://olhovka-90.web.app

Для данного сайта выполняется автоматизация `GitHub`, которая парсит директорию `public` на ресурс `FireBase`. Команда автоматизации находится в файле `deploy.yml`.

Важно! При сборке статического сайта командой `hugo` иногда создается некорректный материал, поэтому перед выполнением этой команды требуется удалить папку `public` и после выполнения команды `hugo` будет создана новая папка `public` с обновлёнными материалами. Также сгенерированный материал должен быть скопирован в папку `docs` для публикации на `GitHub Pages`. 

Чтобы упростить этот процесс с помощью `Shell` скрипта нужно  выполнить в терминале такцю команду:

```sh
rm -rf public/** && hugo && cp -R public/** docs
```

В данной команде сначала удаляется директория `public`, далее выполняется сборка проекта, то есть генерации статического сайта для публикации и сгенерированные материалы копируются в директорию `docs`.

Для того чтобы данная команда работала в терминале, нужно переопределить системную команду `hugo` и создать `Alias` с этой командой в файле рабочей оболочки терминала, например `ZSH` – это файл `.zshrc` . Просто добавьте приведённую ниже строку в файл оболочки.

```sh
alias hugop='rm -rf public/** && hugo && cp -R public/** docs
```

Директория `doсs` должна быть создана заранее.

## Необходимые команды `CLI` для `HUGO`

Создание новой страницыы:

```sh
hugo new newname.md
```

Запуск сервера с отключённым кэшем:

```sh
hugo server -ignoreCashe
```

Запуск сервера с учётом черновиковновиков:

```sh
hugo server -D
```
